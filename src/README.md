# unix and win(windows)

|feature|unix|windows|
|-|-|-|
|Development|Always updated.|Remove some functions.|
|ability|Can do better.|It's not as good as Unix.|
|File support|mp4,m4a,m4v|mp4,m4a|

## Why do we need to separate code between Unix and Windows systems?

Due to different standards, separate code and functionality are required to support both approaches.

## How are they different?

The difference lies in the method of use.

### unix

```bash
get-yt -mp4 dQw4w9WgXcQ
```

### windows

Double-click the .exe file.

```
url / id:  dQw4w9WgXcQ
file (mp4 m4a): mp4
```

And Windows can only download mp4 and m4a files.
